## Beginner Level Exercises

### Exercise 1: System Navigation and Basic Interface

#### Learning Objectives
- Navigate the CLMS interface using keyboard shortcuts
- Locate and identify all 13 main tabs
- Use the global search function
- Access and use the help system

#### Prerequisites
- Basic computer skills
- Understanding of library operations
- Completion of system orientation

#### Materials Needed
- CLMS training environment
- Keyboard shortcut reference card
- Navigation worksheet
- Help system guide

#### Instructions
```
Part 1: Keyboard Shortcut Navigation (15 minutes)

1. Log into the CLMS training environment
2. Practice using the following keyboard shortcuts:
   - Alt+1 through Alt+9 to navigate tabs
   - Ctrl+K to open global search
   - Ctrl+/ to access help
   - F5 to refresh data
   - Esc to exit dialogs

3. Complete the Navigation Challenge Worksheet:
   - Write the keyboard shortcut for each function
   - Test each shortcut and verify it works
   - Note any shortcuts that don't work as expected

Part 2: Tab Exploration (20 minutes)

1. Visit each of the 13 main tabs
2. For each tab, identify and record:
   - Primary purpose of the tab
   - Key features available
   - At least one function you might use regularly
   - Any questions about the tab's purpose

3. Complete the Tab Exploration Worksheet with your findings

Part 3: Search and Help System (15 minutes)

1. Use the global search to find:
   - A student named "Sarah Johnson"
   - A book with "Harry Potter" in the title
   - Equipment available for checkout

2. Access the help system (F1 or Ctrl+/):
   - Search for "barcode scanner setup"
   - Find information about "student management"
   - Locate "circulation procedures" guide

3. Complete the Search and Help Exercise Worksheet
```

#### Evaluation Criteria
```
Navigation Skills (40 points):
• Successfully uses all keyboard shortcuts (20 points)
• Navigates to all 13 tabs correctly (10 points)
• Uses global search effectively (5 points)
• Accesses help system independently (5 points)

Understanding Assessment (20 points):
• Correctly identifies tab purposes (10 points)
• Describes key features accurately (5 points)
• Asks relevant questions about unclear functions (5 points)

Worksheet Completion (40 points):
• Navigation Challenge Worksheet completed accurately (10 points)
• Tab Exploration Worksheet thorough (15 points)
• Search and Help Exercise complete (15 points)
```

#### Debrief Questions
1. Which keyboard shortcut do you think will be most useful in daily operations?
2. Which tab seems most complex and why?
3. What questions do you still have about the interface?
4. How comfortable do you feel navigating the system?

#### Extension Activity
Create a personal quick reference card with the 5 keyboard shortcuts you expect to use most frequently.

### Exercise 2: Student Record Management Basics

- Add new student records to CLMS

- Update basic student information

- Completion of Exercise 1 (System Navigation)
- Understanding of student data requirements
- Familiarity with basic data entry

#### Materials Needed
- CLMS training environment
- Sample student data forms
- Student ID format guide
- Grade category reference chart

#### Instructions
```
Part 1: Adding New Student Records (25 minutes)

You will add 3 new student records using the provided data forms:

Student 1:
- Name: Emily Rodriguez
- Grade: 7
- Section: 7-B
- Email: emily.r@school.edu
- Phone: 555-0123
- Emergency Contact: Maria Rodriguez (Mother)

Student 2:
- Name: James Chen
- Grade: 9
- Section: 9-A
- Email: james.chen@school.edu
- Phone: 555-0456
- Emergency Contact: David Chen (Father)

Student 3:
- Name: Sarah Williams
- Grade: 5
- Section: 5-C
- Email: sarah.w@school.edu
- Phone: 555-0789
- Emergency Contact: Jennifer Williams (Mother)

Follow these steps for each student:
1. Navigate to Student Management (Alt+2)
2. Click "Add Student"
3. Enter the required information in correct format
4. Add the optional information
5. Generate student barcode
6. Save the student record
7. Record the assigned Student ID

Part 2: Student Search Practice (15 minutes)

1. Practice searching for students using different methods:
   - Search by Student ID (use the IDs from Part 1)
   - Search by first name only
   - Search by last name only
   - Search by grade level

   - Note any challenges encountered

Part 3: Student Information Updates (10 minutes)

1. Select one of the students you created
2. Update their information:
   - Add a middle name or initial
   - Update phone number
   - Add a note about library interests
3. Save the changes
4. Verify the updates were saved correctly
```

#### Evaluation Criteria
```
Data Entry Accuracy (40 points):

Search Skills (25 points):
• Successfully located all created students (10 points)
• Used multiple search methods effectively (10 points)
• Completed search practice worksheet thoroughly (5 points)

Update Procedures (20 points):
• Successfully updated student information (10 points)
• Changes saved correctly (5 points)
• Verified update accuracy (5 points)

Barcode Generation (15 points):
• Generated barcodes for all students (10 points)
• Barcodes printed and formatted correctly (5 points)

- Barcode scanner
- Sample book records
- Due date calculation guide
- Fine structure reference

```
Part 1: Standard Checkout Procedures (20 minutes)

You will process 5 book checkout transactions:

Transaction 1:
- Student: Emily Rodriguez (Grade 7)
- Book: "The Hunger Games" (ACC001234)
- Due Date: Standard 14-day period

Transaction 2:
- Student: James Chen (Grade 9)
- Book: "To Kill a Mockingbird" (ACC001235)
- Due Date: Standard 14-day period

Transaction 3:
- Student: Sarah Williams (Grade 5)
- Book: "Charlotte's Web" (ACC001236)
- Due Date: Standard 14-day period

Transaction 4:
- Student: Emily Rodriguez (Grade 7)
- Book: "Harry Potter and the Sorcerer's Stone" (ACC001237)
- Due Date: Standard 14-day period

Transaction 5:
- Student: James Chen (Grade 9)
- Book: "1984" (ACC001238)
- Due Date: Standard 14-day period

Follow these steps for each transaction:
1. Enter or scan student ID
2. Verify student account status
3. Enter or scan book accession number
4. Verify book availability
5. Confirm due date calculation
6. Process checkout
7. Print receipt if requested

Part 2: Book Return Procedures (20 minutes)

You will process 3 book returns:

Return 1:
- Book: "The Hunger Games" (ACC001234)
- Return Date: 5 days after checkout
- Condition: Good (no new damage)

Return 2:
- Book: "To Kill a Mockingbird" (ACC001235)
- Return Date: 16 days after checkout (overdue)
- Condition: Good (no new damage)

Return 3:
- Book: "Charlotte's Web" (ACC001236)
- Return Date: 7 days after checkout
- Condition: Minor damage (small tear on cover)

Follow these steps for each return:
1. Enter or scan book accession number
2. Verify checkout information
3. Determine if book is overdue
4. Assess book condition
5. Select appropriate condition option
6. Add condition notes if needed
7. Calculate fines if applicable
8. Process return

Part 3: Fine Calculation Practice (10 minutes)

1. Calculate fines for the following scenarios:
   - Book 2: 2 days overdue at $0.10 per day
   - Book 3: Minor damage fee of $2.00
   - If both scenarios happened to the same student, what would be the total fine?

2. Complete the Fine Calculation Worksheet showing your work
```

• Book identification completed correctly (10 points)

• Overdue calculation accurate (5 points)
• Return procedures followed correctly (10 points)

Fine Calculation (15 points):
• Overdue fines calculated correctly (5 points)
• Damage fees applied appropriately (5 points)
• Total fines calculated accurately (5 points)

Customer Service (10 points):
• Professional communication maintained (5 points)
• Clear explanations provided to patrons (5 points)
```

#### Debrief Questions
1. What was the most challenging part of the checkout process?
2. How would you handle a situation where a book is severely damaged?
3. What questions might patrons have about fines or due dates?
4. How confident do you feel processing circulation transactions?

Practice processing a checkout for a student who has overdue books and needs to pay fines before checking out additional items.

## Intermediate Level Scenarios

### Scenario 1: Complex Circulation Situations

#### Learning Objectives
- Handle multiple book checkout with mixed due dates
- Process reserves and holds
- Manage account restrictions and limitations
- Handle lost or damaged book situations

#### Prerequisites
- Completion of all beginner exercises
- Understanding of circulation policies
- Experience with basic checkout/return procedures

#### Scenario Setup
```
Background Information:
- It's the week before final exams
- The library is very busy
- Multiple patrons need assistance simultaneously
- Several complex situations require attention

Characters/Profiles:
1. Teacher (Ms. Thompson): Needs to reserve 10 books for class project
2. Student (Alex Martinez): Has 3 overdue books and wants to check out more
3. Parent (Mrs. Davis): Reporting lost book and needs to pay replacement cost
4. Student (Sam Johnson): Wants to reserve a popular book that's currently checked out

Materials Needed:
- CLMS training environment with complex scenario data
- Reserve request forms
- Fine calculation worksheets
- Lost book processing forms
```

#### Scenario Instructions
```
Situation 1: Teacher's Bulk Reserve Request (15 minutes)

Ms. Thompson needs 10 copies of "The Great Gatsby" for her English class next week.

Your tasks:
1. Check availability of "The Great Gatsby" (ACC001239)
2. Determine how many copies are available now
3. Check if any copies are due back before next week
4. Process reserve requests for unavailable copies
5. Schedule pickup for available copies
6. Provide Ms. Thompson with reservation details
7. Document the request in the system notes

Situation 2: Student with Account Restrictions (15 minutes)

Alex Martinez (Grade 11) wants to check out 3 books but has restrictions:
- 2 overdue books (5 days overdue)
- $5.25 in unpaid fines
- Maximum checkout limit: 5 books

Your tasks:
1. Look up Alex's account and explain the restrictions
2. Calculate additional fines for overdue books
3. Explain options for resolving the account issues
4. Process payment if Alex decides to pay fines
5. Process return of overdue books if Alex has them
6. Complete the new checkout if issues are resolved
7. Document the interaction in the system

Situation 3: Lost Book Processing (10 minutes)

Mrs. Davis is reporting that her daughter lost "Charlotte's Web" (ACC001240)
- Original cost: $15.99
- Lost 30 days ago
- Already overdue by 15 days when lost

Your tasks:
1. Verify the book is checked out to Mrs. Davis's daughter
2. Calculate total charges:
   - Overdue fines (15 days × $0.10 = $1.50)
   - Replacement cost ($15.99)
   - Processing fee ($2.00)
3. Mark book as lost in the system
4. Generate bill for Mrs. Davis
5. Explain payment options
6. Update the student's account
7. Remove the lost book from active circulation

Situation 4: Popular Book Reservation (10 minutes)

Sam Johnson (Grade 10) wants to reserve "Divergent" (ACC001241)
- Currently checked out to another student
- Due back in 3 days
- 3 other students are on the waitlist

Your tasks:
1. Check the current status of "Divergent"
2. Explain the reservation process to Sam
3. Add Sam to the waitlist
4. Provide estimated availability date
5. Explain notification system for reservations
6. Document Sam's reservation preferences
```

#### Evaluation Criteria
```
Complex Problem Solving (40 points):
• Handles multiple simultaneous situations (10 points)
• Prioritizes tasks effectively (10 points)
• Applies system knowledge correctly (10 points)
• Maintains composure under pressure (10 points)

Policy Application (30 points):
• Applies circulation policies correctly (15 points)
• Explains policies clearly to patrons (10 points)
• Makes appropriate judgment calls (5 points)

Customer Service (20 points):
• Maintains professional communication (10 points)
• Provides clear explanations and options (10 points)

System Utilization (10 points):
• Uses advanced system features appropriately (5 points)
• Documents interactions thoroughly (5 points)
```

#### Debrief Questions
1. How did you prioritize the multiple situations?
2. What was the most challenging scenario and why?
3. How would you handle these situations differently if they occurred during regular library hours?
4. What additional system features would be helpful for these scenarios?

#### Extension Activity
Practice a scenario where all four situations occur simultaneously, and you need to prioritize and delegate tasks appropriately.

- Students
- Parents
- Library staff
- Community members

- Budget allocation worksheets
- Staffing scenarios
